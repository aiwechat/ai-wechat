# Agent Notes

本项目是融合 AI 智能助手的分布式即时聊天系统。Codex 或其他协作者修改代码时，请优先遵守以下约定。

## 协作规则

- 不直接向 `main` 提交代码；成员 C 的开发分支建议为 `feature/protocol-db`。
- 小步提交，提交信息要说明具体功能，例如 `实现协议帧编解码`。
- 合并主分支前先同步最新代码，遇到冲突先沟通，不盲目删除他人逻辑。
- 本项目第一阶段以协议和数据库稳定为优先，基础聊天功能可联调后再扩展 AI、审核、文件传输。

## 安全与仓库卫生

- 不在代码中写死 API Key、密码、Token。
- 不提交 `.env`、数据库文件、日志、缓存、上传文件、接收文件。
- 可提交 `.env.example`，但其中只能放占位符。
- 数据库 schema 有变化时，必须同步说明迁移影响。

## 代码约定

- Python 代码优先使用标准库，除非全组确认新增依赖。
- JSON 字段名使用小写加下划线，例如 `request_id`、`group_id`。
- 协议错误统一返回 `error` 消息，payload 中包含 `error_code`、`message`、`detail`。
- 服务端和客户端都应通过 `common/protocol.py` 编解码消息，避免重复拼 JSON。
- 服务端访问 SQLite 时应通过 `server/database.py`，避免在路由层手写 SQL。

## 文档同步

- 协议字段、消息类型、错误码发生变化时，必须同步更新 `docs/协议说明.md`。
- 新增成员 D/E 的 AI、审核、文件传输能力时，优先扩展 `payload` 或 `meta`，不要随意破坏 Envelope 顶层结构。
- README 面向组内成员解释设计思路，协议说明面向客户端和服务端联调使用。
