使用场景

典型交互流程

以下是几个代表性的使用场景，展示系统的核心交互能力。

🎬 录制：注册登录与私聊 保存为 gifs/login-chat.gif
🔐

注册 → 登录 → 聊天

用户注册账号后登录，查看在线用户列表。选择目标用户发起私聊，消息实时送达。 若对方离线，消息自动持久化，待对方上线后推送。

🎬 录制：群组创建与群聊 保存为 gifs/group-chat.gif
🏗️

创建群组 → 邀请成员 → 群聊

创建群组获取群 ID，分享给其他用户加入。群内消息广播送达所有成员。 支持 @AI 触发智能问答，上传图片进行多模态分析。

🎬 录制：图片上传与 AI 多模态 保存为 gifs/ai-vision.gif
📸

发送图片 → 多模态分析

选择图片上传至聊天，服务端接收并持久化。在群聊中使用 @AI 配合图片， AI 助手自动分析图片内容并返回图文解读。

🎬 录制：消息撤回与审核 保存为 gifs/recall-moderation.gif
🔄

消息撤回 → 内容审核

发送消息后可一键撤回。同时系统自动审查每条消息内容， 违规消息由服务端强制撤回并警告发送者，保障聊天环境安全。

🎬 录制：多端互通 保存为 gifs/multi-device.gif
📱

多端互通

CLI 命令行客户端与浏览器 GUI 通过统一中继服务接入同一会话空间， 共享在线用户表和聊天历史，两边的用户可以互相实时私聊和群聊。

🎬 录制：文件传输 保存为 gifs/file-transfer.gif
📂

文件分享

选择任意文件上传至聊天，系统自动分片传输（每片 64KB）。 服务端落盘保存，接收方通过受控链接下载，支持最大 50MB 文件。

核心能力

功能一览

从注册登录到 AI 助手，覆盖即时通讯的完整链路。所有功能通过统一协议协同工作，CLI 与浏览器客户端共享同一会话空间。

👤

用户注册与登录

密码使用 PBKDF2 加盐哈希安全存储。支持单点登录，新连接自动踢出旧会话，防止账号共享。

💬

一对一私聊

在线消息实时转发，离线消息持久化落盘。每条消息附带回执确认，确保可靠投递。

👥

群组聊天

支持创建群组、加入群组、退出群组和查看成员列表。消息在群内广播，新成员可查看历史记录。

🤖

AI 智能助手

在聊天中输入 @AI 即可与 AI 对话。支持纯文本问答与多模态图片分析，配置 OpenAI 兼容 API。

📎

文件传输

支持图片、语音、通用文件分片上传（file_start → file_chunk → file_end）。服务端落盘保存，通过受控链接下载。

↩️

消息撤回

发送者可一键撤回已发送的私聊或群聊消息，服务端广播撤回通知并在历史记录中隐藏原内容。

🛡️

内容安全审查

消息先正常发送，服务端异步调用 API 审查。违规消息自动强制撤回并向发送者发出警告。无 API 时回退本地关键词过滤。

🟢

在线状态广播

用户上下线时实时广播 USER_STATUS 消息，所有在线用户即刻看到状态变化。同步展示当前在线人数。

💓

心跳检测

服务端后台线程定期扫描连接，超时未响应的连接自动清理并广播离线状态，保持在线用户列表准确。

📜

聊天历史

服务端 SQLite 持久化所有消息。客户端可随时拉取任意会话的历史记录，本地缓存仅用于当前会话显示。

🖥️

多端互通

CLI 命令行客户端与浏览器 GUI 通过统一中继服务共享在线会话表，互相实时私聊、群聊、同步在线状态。

🌓

日夜主题

支持浅色与深色双主题，自动跟随系统偏好，手动切换后本地持久化。移动端与桌面端体验一致。

快速开始

启动服务

一行命令启动统一中继服务，同时支持 CLI 客户端和浏览器 GUI，两端共享在线用户和聊天历史。

# 启动统一中继服务（TCP + Web）
python3 -m server.relay_service \
  --tcp-host 0.0.0.0 --tcp-port 9000 \
  --web-host 0.0.0.0 --web-port 8080

# CLI 连接
python3 -m client.ui --host 127.0.0.1 --port 9000

# 浏览器访问
# 打开 http://127.0.0.1:8080

配置 .env 文件即可启用 AI 助手和安全审查，详情见 README。

技术栈

核心技术

项目采用 Python 全栈实现，通信协议自定义设计。

通信协议

4 字节长度前缀 + UTF-8 JSON 应用层帧格式，统一 ProtocolMessage 消息结构。支持粘包 / 拆包处理。

TCPWebSocketJSON自定义协议

数据持久化

SQLite 轻量级数据库，存储用户账户、群组信息、聊天记录和文件传输状态。密码 PBKDF2 加盐哈希。

SQLitePBKDF2CRUD

并发模型

每连接一线程处理帧收发，后台心跳扫描独立线程。支持 60+ 客户端并发连接压力测试。

threadingsocketasyncio

AI 集成

OpenAI 兼容 API 接口，支持 MiMo、GPT-4o 等模型。@AI 触发智能问答，支持多模态图片分析。

OpenAI APIMiMoGPT-4o

前端

纯原生 HTML / CSS / JavaScript，无框架依赖。响应式布局适配手机与桌面，支持深色主题切换。
