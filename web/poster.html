基于 TCP 与 AI API 的分布式即时聊天系统设计

AI-WeChat: Multi-platform Real-time Chat with Unified Relay, AI Assistant and Moderation

代码库逻辑抽象：客户端入口 → 帧协议传输 → 统一中继 → 消息路由 → 数据持久化 / AI / 审核 / 文件服务

Project Summary

系统覆盖注册登录、在线状态、私聊、群聊、群组管理和历史拉取。

CLI 与 Web GUI 共用统一中继核心，实现跨平台实时互通。

扩展 AI 助手、API 内容审核、强制撤回、图片/语音和文件分片传输。

总体亮点：多平台入口统一进入 Relay，路由层集中调度聊天、AI、审核、文件与数据库能力。
1

多平台接入与统一中继

CLI、桌面浏览器与手机浏览器进入同一个 Relay，共享在线会话、群组状态和历史数据。

2

用户会话与即时通信核心

消息路由集中处理注册登录、私聊、群聊、群组操作、心跳和历史请求。

3

协议、存储与消息增强

统一 Envelope 把 TCP 帧、WebSocket 文本、数据库记录和前端展示连接成一条稳定链路。

4

AI 助手与内容安全审查

@AI 回复和内容审核都放入后台线程池，保证聊天主流程先响应，再异步补充智能能力。

5

Evaluation: 模块测试与系统验证

从单元测试、集成测试到 60 客户端并发压力测试，验证协议、路由、文件、AI 与审核链路的稳定性。

6

Future Work: 后续改进方向

在现有统一中继、AI 助手、安全审查和文件传输基础上，进一步增强智能性、可治理性和可扩展性。

01
AI context memory & streaming replies加入群聊上下文摘要、连续对话记忆和流式输出，提升 AI 交互体验。
02
Fine-grained moderation policy支持分级规则、群管理员配置、人工复核和安全审计日志。
03
Resumable file transfer扩展断点续传、失败重试、秒传校验和下载进度管理。
04
Scalable deployment & monitoring引入 PostgreSQL、Redis Pub/Sub、异步 I/O、日志追踪和连接监控。
